Notes on Using LabView for DAQ


(Last Updated December 14, 2017, by Eric Prebys)

These notes apply to Windows.  I've had less luck running LabView on MacOS, but it works with the Windows partition on my Mac.

Required Software

It's sometimes difficult to figure out what software you need for LabView DAQ, but the only two pieces you need are
  • LabView
    • Select "Measure Physical Systems..." option
    • Currently using LabView2017
    • You can get a 7 day free trial version, but extend it to 45 days by creating an account at NI.
  • DAQmx
    • This has the stuff that communicates with hardware, as well as the all-important "DAQ Assistant"
No additional software or drivers should be needed to use hardware with LabView

Connecting to Hardware

I've tested LabView with a USB-6000 USB-based module and with a cDAQ-9184 four-slot ethernet-based chassis, populated with
  • 1 x NI-9403 32 channel digital IO module
  • 2 x NI-9205 32 channel analog input module
  • 1 x NI-9201 8 channel analog input module
When the USB device is connected, the computer should recognize it and download the appropriate drivers. 

The ethernet device should just be "discovered" on the local network. 

Hardware is managed with the NIMax program, which is installed along with DAQmx.  When you run the progam, you should see USB devices under "My Computer->Devices and Hardware", and your ethernet devices under "My Computer->Devices and Hardware->Network Devices".  If it doesn't see it automatically, you may need to go to the "...Network Devices" page, and click "Add Network Device" at the top.  This should see any networked devices, which you can then click on and select "Add".  Once you add it, it will recognize any modules as separate devices and load the appropriate drivers.

Incorporating DAQ Hardware into LabView Designs


The magic tool for inserting DAQ channels into LabView designs is "DAQ Assistant", this is available on the block diagram with "(Right-click)->Measurement I/O->NI DAQmx->DAQ Assistant".  The first time you use this, you may get an message that your software is unlicensed, so you have to click the box that says you agree to follow the licensing rules, and click next.  Then you will get a pop-up window that allows you to select "Aquire Signals" or "Generate Signals", and then select analog or digital input or output (or a bunch of other stuff you probably won't use).

Troubleshooting

Issues with Running a VM

  • As usual, you have to tell the computer to connect the USB device to the Windows VM when you plug it in
  • It seems to require that the Windows VM have direct access to the network, rather than via the default "shared network".  This is done by going to "Devices->Network1->LAN (or whatever)".  This doesn't appear to affect anything else

Trouble with Ethernet Connection

If NIMax doesn't see the ethernet device, check the yellow status light.  It should be off.  It is repeating a sequence of three flashes, it means the firmware is corrupted, and you need to re-flash it by following the instructions here.

Moving the Ethernet Device from One Host to Another

Ethernet devices can only be used by one host at a time.  If you try to connect to it with another (at least on the same network), you will get an error in NIMax.  The solution is to right-click on the device in the left-hand menu and select "Reserve this Chassis".

Notes on Updating MacOS

El Capitan (10.11) to Sierra (10.12)

General

I had a very bad experience with a Fermilab "official version" of an OS, so since then I always just use the vanilla Apple install from the App store.

Installation went remarkably quickly compared to some of the previous upgrades, and there are no more big changes to access rules that cause so much problems a couple of versions ago, so there shouldn't be big problems. 

Here are some specific issues afterwards...

Cisco AnyConnect VPN Client

Before you upgrade, make sure you have the latest version! (3.1.04063).  I had an older version, which the upgrade broke.   If you're at home, make sure you use the old version to download the new version, because you won't be able to later.

Things that Worked Fine

  • All the normal stuff (browsers, editors, Office, etc)
  • TexWorks. I already did the following workaround from the previous upgrade
    • Go into "Preferences->typesetting" and add "/Library/TeX/texbin/" to the search path. 
  • Parallels 11
  • MADX
  • ACNET
I don't run local versions of MARS, ART, GEANT4, or Synergia at this time.

ROOT

It's always kind of a pain to get ROOT to work properly after an upgrade.  This is because the update upgrades Xcode, but not the command line tools.  To fix this, you need to:
  • Launch XCode
    • It will tell you it needs to download and install components.  Let it do so
    • Quit
  • Install the command line tools
    • If you do the recommended checks, it will tell you they're already there, but this is a lie.
    • Go here. Log in with your Apple credentials
    • Download the latest command line tools
    • Install them
  • Download the latest ROOT from here
  • To be on the safe side, delete any .so files in your area and let it rebuild them.

XQuartz/ssh

After the upgrade, I found that ssh no was no longer forwarding X11 connections.  Reinstalling XQuartz fixed the problem.

g4Beamline/histoRoot

The g4beamline executables work fine.  The visualizer has a bug that won't work with XQuartz newer than 2.7.8, which still works under Sierra.  Tom Roberts is working on a fix.

histoRoot works fine

MathType

MathType for Mac continues to only work with Office 2011.  I found that the first time I used it after the upgrade, it had to download a bunch of fonts (around five).  After that it worked fine.

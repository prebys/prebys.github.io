Opening X11 Windows on an MS Windows PC


If you're going to interact a lot with Unix, it's probably best to install Cygwin and Cygwin X, but that's overkill for most people. If the goal is just to be able to log into a Fermilab unix node and open X11 windows on your MS Windows machine (eg if you need to run ROOT), the easiest way I've found is described below.

Installing software

  • Install Kerberos
    • Note! If you already have Kerberos on your system, use that!  Don't install a second version, because they can interfere with each other.  Otherwise...
    • Download and install the latest kerberos tools from MIT
    • Download the appropriate krb5.conf file for Fermilab and save it to "C:\ProgramData\Kerberos" (overwrite the file that is there).
  • Install and configure Xming and Putty
    • Follow the instructions here to install and configure Xming and Putty, with the following addition
      • Before saving the configuration file in Putty, go to the "Connection->SSH->Auth->GSSAPI" screen and make sure both radio button options at the top are selected.
    • Important note:  If you want to save separate Putty configurations to connect to multiple nodes, it always starts with the default configuration, so you have to do all the configuration steps for each one, rather than simply change the node name.

Starting a session

  • Start the X server by double clicking the "Xming" icon.  There will be no splash screen or any other indication that you have done this. Doing it more than once is OK if you're not sure it's running.
  • Open a command window get a kerberos ticket by typing:
    • > kinit -f (your principle)@FNAL.GOV     (capitals are important!)
  • Double click on the Putty icon, then double-click on the appropriate configuration name
    • This should open a window on the approriate linux node with correct X11 tunneling.  To test, try popping an Xwindow by typing
      • > xterm &

Notes on Java Files to Study AC Dipole Transmission for Mu2e

(last updated April 9, 2014)

These are notes on a suite of Java files I wrote to study details of bunch transmission through the Mu2e beam line extinction system

Working Area

All source, input, and output files are kept in /Users/prebys/GoogleDrive/eclipse/workspace

Main Classes

The study is based on three major classes:
  • BeamBunch (abstract):  This class returns a longitudinal probability distribution function (PDF) for a bunch. Derived classes
    • BeamBunchGaussian: Gaussian bunch
    • BeamBunchLookup: Bunch based on a lookup table.  Current best lookup table is time_distribution_new.csv
  • KickerWave (abstract): This class returns a wave function corresponding to the deflection field of the AC diple.  In general, it's normalized to return a value of 2 (full extinction) at the boundaries of a specified transmission window. Derived classes:
    • KickerSineWave: simple sine wave
    • KickerModifiedSineWave:  sine wave plus a higher harmonic.  This is the base line for Mu2e.
    • KickerMECOWave: Original MECO waveform
    • KickerUCIWave: Later proposed alternative by UCI
  • Aperture:  This defines the aperture. It returns a transmission probability as a function of normalized amplitude, where 1 corresponds to the beam being centered on the edge of the collimator. For backward compatibility, it has two modes, based on which constructor is used
    • Gaussian mode: If constructed with Aperture(double emittance95/admittance), it will create a Gausian transverse profile, based on the ration between the 95% emittance and the beam admittance.
    • Lookup mode: If constructed with Aperture(File lookupFile), it will look up the transmission as a function of normalized displacement.  Current best transmission file is transmission_lookup.csv.
There are a number of analysys files using these classes. Their behavior can generally be inferred from the name.
